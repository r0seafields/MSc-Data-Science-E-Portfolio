Machine Learning

Unit 6 and Unit 11
Initial Proposal and Final Project

Overview

This development team project provided experience of the complete machine learning lifecycle, from designing an initial proposal to implementing and evaluating multiple deep learning models. The project investigated whether Airbnb could use structured, text-based and spatial data to identify high-demand listings in New York City and support business decision-making.

Evaluation

The initial proposal established a clear business question and outlined a deep learning approach using four models: a tabular neural network, an LSTM sentiment model, a CNN map-based model and a hybrid model combining all three approaches. It also identified review activity as a practical proxy for demand because booking data was unavailable.

The final project successfully implemented the proposed methodology while refining the data preparation and feature engineering following exploratory analysis. Model evaluation demonstrated that the tabular neural network produced the strongest overall performance, while the sentiment model provided useful additional insight from listing titles. In contrast, the CNN map model contributed little predictive value, highlighting that increasing model complexity does not necessarily improve predictive performance.

Comparing the proposal with the completed project demonstrated the importance of validating assumptions through experimentation and selecting models based on evidence rather than complexity. Working on the project strengthened my skills in data preprocessing, feature engineering, neural network development, model evaluation and communicating technical findings through practical business recommendations.
