Unit 8 – Gradient Descent

Introduction

This unit introduced gradient descent, one of the most important optimisation algorithms used in machine learning. Gradient descent works by iteratively adjusting model parameters to minimise a cost function, allowing a model to learn the relationship between input data and target values. Through practical experimentation in Google Colab, I investigated how changing the learning rate and the number of iterations affected the optimisation process and model convergence.

Practical Activity

I completed the Gradient Descent practical exercise using Google Colab, following the KDnuggets tutorial by Mayo (2017). During the activity I experimented with different learning rates and iteration values to observe their effect on the reduction of the cost function during model training.

Reflection

This activity helped me develop a much stronger understanding of how gradient descent optimises machine learning models. By changing the learning rate and the number of iterations, I was able to observe how these hyperparameters influence the speed and effectiveness of convergence. Increasing the number of iterations reduced the cost function further, although improvements became smaller as the algorithm approached the optimum solution. A smaller learning rate produced slower but stable learning, whereas larger learning rates reduced the cost more quickly. For this dataset, the default learning rate produced the lowest final cost after 100 iterations, demonstrating the importance of selecting appropriate hyperparameters rather than simply increasing their values. This practical activity reinforced the theoretical concepts discussed in the module and improved my understanding of optimisation within supervised learning algorithms.
